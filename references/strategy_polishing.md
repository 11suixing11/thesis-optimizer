# 学术润色策略

## 概述

本文档为 `polishing_agent` 提供详细的学术润色策略。

## 核心原则

- 润色是锦上添花，不是重写
- 不改变核心观点和论证逻辑
- 不添加原文没有的信息
- 优先处理影响理解的问题

## 策略一：表达精准化

### 1.1 量化抽象概念

| 模糊表述 | 精准化 |
|---------|--------|
| 效果较好 | 准确率达到95.3% |
| 近年来 | 2018年以来 |
| 大量研究 | 超过50项研究 |
| 显著提升 | 提升幅度达40% |
| 具有重要意义 | 对XX领域的发展具有推动作用 |

### 1.2 明确因果关系

- 原文：「该因素对结果有影响」
- 润色：「该因素与结果之间呈正相关关系（r=0.72, p<0.01）」

### 1.3 减少模糊表述

- 原文：「可能」→「在XX条件下，概率为XX%」
- 原文：「一定程度上」→「在XX范围内」
- 原文：「相关」→「呈XX相关（r=XX）」

## 策略二：学术规范性

### 2.1 术语一致性

**检查项**：
- 全文是否混用同一概念的不同表述
- 缩写首次出现是否给出全称
- 术语首次出现是否给出定义

**示例**：
- 问题：混用"深度学习"和"深层学习"
- 修复：统一使用"深度学习"，首次出现时给出定义

### 2.2 时态规范

| 场景 | 时态 | 示例 |
|------|------|------|
| 文献综述 | 过去时 | 张三(2020)提出了... |
| 方法描述 | 过去时 | 本研究采用了... |
| 结果呈现 | 过去时 | 实验结果显示... |
| 讨论分析 | 现在时 | 这表明... |
| 普遍事实 | 现在时 | 深度学习是... |

### 2.3 引用格式

**检查项**：
- 文内引用格式是否统一
- 文内引用与参考文献列表是否一一对应
- 引用年份、作者名是否准确

## 策略三：可读性优化

### 3.1 复杂句拆分

**原则**：超过60字的长句考虑拆分。

**示例**：
- 原文：「本文提出了一种基于深度学习的图像识别方法，该方法通过引入注意力机制和多尺度特征融合技术，显著提升了模型在复杂场景下的识别准确率。」
- 拆分：「本文提出了一种基于深度学习的图像识别方法。该方法引入了注意力机制和多尺度特征融合技术。在复杂场景下，模型的识别准确率得到了显著提升。」

### 3.2 过渡流畅性

**原则**：不使用"首先其次最后"的机械过渡，使用自然的语义衔接。

**技巧**：
- 使用代词衔接：「这一发现...」「该方法...」
- 使用语义关联：「在此基础上...」「进一步...」
- 使用逻辑连接：「因此...」「然而...」

### 3.3 信息层次

**原则**：重要信息前置。

**技巧**：
- 论点在前，论据在后
- 结论在前，推导在后
- 新信息在前，旧信息在后

## 执行流程

1. 读取论文全文或指定章节
2. 按三个维度逐一检查
3. 执行润色修改
4. 生成修订日志
5. 标记完成状态
