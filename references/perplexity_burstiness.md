# 句式节奏与统计风险提示

## 概述

本文档定义了用于评估文本"人类感"的统计指标，指导 `ai_reduction_agent` 的优化方向。

## 核心指标

### 1. 句长标准差（Sentence Length Std Dev）

**定义**：段落中各句长度（字符数）的标准差。

**目标值**：> 8

**风险阈值**：
| 标准差 | 风险等级 | 说明 |
|--------|---------|------|
| < 3 | 高风险 | 句式高度一致，明显AI特征 |
| 3-5 | 中风险 | 句式较规整，有AI嫌疑 |
| 5-8 | 低风险 | 基本正常 |
| > 8 | 安全 | 句式多样，人类特征明显 |

**优化策略**：
- 短句（< 15字）：适当扩展，增加细节
- 长句（> 40字）：拆分为2-3个短句
- 等长句：故意制造长短差异

### 2. 词汇丰富度（Type-Token Ratio, TTR）

**定义**：不同词汇数 / 总词汇数。

**目标值**：> 0.6

**风险阈值**：
| TTR | 风险等级 | 说明 |
|-----|---------|------|
| < 0.4 | 高风险 | 词汇重复严重 |
| 0.4-0.5 | 中风险 | 词汇较单一 |
| 0.5-0.6 | 低风险 | 基本正常 |
| > 0.6 | 安全 | 词汇丰富 |

**优化策略**：
- 替换高频重复词为同义词
- 使用专业术语的多种表述
- 增加形容词和副词的变化

### 3. 困惑度（Perplexity）

**定义**：衡量文本的可预测性。AI生成的文本通常困惑度较低（太规整），人类写作困惑度较高（有随机性）。

**目标值**：无固定值，但应高于AI生成文本的典型困惑度。

**优化策略**：
- 引入适度的"意外"元素
- 使用非常规的句式结构
- 保留人类写作的不完美性

## Burstiness（突发性）

**定义**：衡量文本中信息密度的变化。人类写作通常在某些段落信息密集，某些段落较松散。AI写作则倾向于信息密度均匀。

**优化策略**：
- 在重要论点处增加信息密度
- 在过渡处适当降低信息密度
- 模拟人类写作的"起伏"感

## 实际应用

### 优化前检查

```
1. 计算目标段落的句长标准差
2. 计算目标段落的TTR
3. 标记低于阈值的段落
4. 按风险等级排序
```

### 优化后验证

```
1. 重新计算优化后段落的指标
2. 确认所有指标达到目标值
3. 如果未达标，继续优化
```

## 注意事项

1. 这些指标是参考，不是绝对标准
2. 学术论文本身有一定的规整性要求
3. 过度追求指标可能导致文本质量下降
4. 需要结合具体学科特点灵活应用
