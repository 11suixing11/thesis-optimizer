# AI 写作风险特征降低策略

## 概述

本文档为 `ai_reduction_agent` 提供详细的降AI检测策略。

## 核心原则

### ⚠️ IRON RULE

- 修改范围严格限制在单句内
- 绝不大段推翻重写
- 保留原始逻辑和思维跳跃
- 专业术语永远保护原文

## 策略一：句内重组

### 1.1 主谓宾顺序调整

**原始**：[主语] + [谓语] + [宾语]
**重组**：[状语] + [主语] + [谓语] + [宾语]

**示例**：
- 原文：「本研究提出了一种新的方法」
- 优化：「针对该问题，本研究给出了一种新的求解方法」

### 1.2 因果关系重组

**原始**：因为A，所以B
**重组**：B的出现，可归因于A

**示例**：
- 原文：「由于数据量不足，导致模型精度下降」
- 优化：「模型精度的下降，与数据量不足密切相关」

### 1.3 被动语态转换

**原始**：主语 + 谓语 + 宾语（主动）
**转换**：宾语 + 被 + 主语 + 谓语（被动）

**示例**：
- 原文：「本文验证了该方法的有效性」
- 优化：「该方法的有效性在本文中得到了验证」

## 策略二：语气自然化

### 2.1 添加适度主观性

- 原文：「实验结果表明该方法优于现有方法」
- 优化：「从实验结果来看，该方法相比现有方法具有一定优势」

### 2.2 保留不确定性

- 原文：「该方法能够解决该问题」
- 优化：「该方法在一定程度上可以缓解该问题」

### 2.3 适度口语化

- 原文：「该方法具有较高的效率」
- 优化：「简单来说，该方法在效率方面表现不错」

## 策略三：打破规整化

### 3.1 连接词替换

| AI典型用法 | 替换策略 |
|-----------|---------|
| 首先...其次...最后 | 删除序数词，用语义衔接 |
| 不仅...而且... | 简化为直接陈述 |
| 一方面...另一方面... | 拆分为独立句子 |
| 综上所述 | 删除或改为具体总结 |

### 3.2 句式多样化

**目标**：段落中至少有3种不同长度的句子。

**技巧**：
- 插入短句（< 15字）打破节奏
- 将长句拆分为短句群
- 偶尔使用反问或设问

### 3.3 结构打破

**AI典型结构**：论点 → 论据 → 总结（每段都如此）

**打破策略**：
- 偶尔省略总结句
- 先给论据再给论点
- 在段落中插入个人见解

## 策略四：消除AI高频词

参照 `references/ai_vocabulary_blacklist.md`，对高风险词汇进行替换或删除。

## 执行流程

1. 识别目标段落的AI特征
2. 选择对应的策略
3. 执行句内微调
4. 验证语义一致性
5. 记录修订日志
